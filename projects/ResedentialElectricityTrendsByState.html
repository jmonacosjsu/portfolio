Residential Electricity Consumption Per Capita (2010–2024)

1.0 Introduction

Residential electricity use looks very different depending on which state you're looking at and when. Since states vary so much in population size, raw consumption totals don't tell you much on their own. A high-population state will almost always use more electricity in total, regardless of individual behavior or efficiency.

This analysis focuses on residential electricity consumption per capita: which states saw usage rise or fall over time, and whether those shifts happened gradually or clustered around specific periods like 2020.

No causal claims are made here. Weather, policy, and behavior all play a role in shaping consumption patterns, but untangling those factors is beyond the scope of this project. The per-capita framing simply levels the playing field across states, making it possible to compare trends that would otherwise be obscured by population differences. That context is useful for benchmarking and thinking about energy efficiency, even without pinning outcomes on specific causes.

2.0 Sources of Data

2.1 Electricity Sales Data

Residential electricity sales data were obtained from the U.S. Energy Information Administration (EIA). The dataset includes annual retail sales of electricity to ultimate consumers, disaggregated by sector and by state.

Key fields include: Year · State · Sales (MWh)

2.2 Population Data

State population data were sourced from the United States Census Bureau's State Population Totals for the years 2010–2024.

Key fields include: Year · State · Population

These datasets were selected to allow residential electricity consumption to be normalized on a per-capita basis, enabling consistent comparisons across states and over time.

3.0 Process: Data Cleaning, Normalization & Validation (Python)

3.1 Data Cleaning

Prior to processing in Python, both datasets were briefly reviewed in Google Sheets to verify structure, remove non-data headers, and confirm consistent formatting of year and state fields. The datasets were then imported into Python using pandas.

The primary data-cleaning steps included:

  • Standardizing column names and data types: Column names were normalized (lowercased in the EIA file) and renamed to consistent fields used throughout the workflow (state, state_code, year, sales_mwh, population).
  • Aligning state identifiers: State names from the Census file were cleaned (leading characters removed) and mapped to two-letter state codes to enable a reliable join key.
  • Reshaping population data: Census population data were converted from wide format (years as columns) to long format (one row per state-year).
  • Filtering to comparable geography: The EIA "US" aggregate row was removed so the dataset reflects state-level observations only.

3.2 Normalization

After cleaning, the population and EIA sales datasets were merged on state_code and year. Electricity sales were converted from MWh to kWh, and per-capita consumption was computed as kWh per person for each state-year.

3.3 Validation Checks

Basic validation checks were used to confirm the merged dataset behaved as expected (e.g., confirming year bounds, expected state counts, and verifying that the merge produced the expected row count). These checks support internal consistency before visualization in Tableau.

3.4 Jupyter Notebook Processing Steps

Data processing was performed in a single Jupyter Notebook using Python and the pandas library. The notebook focused on reshaping the population data, aligning state identifiers across datasets, merging population and electricity sales data, and computing residential electricity consumption per capita.

Population data from the U.S. Census Bureau were first cleaned by standardizing the state column, removing non-data characters, and reshaping the dataset from wide format to long format. State names were then mapped to two-letter state codes to create a consistent join key.

Residential electricity sales data from the EIA were imported separately, column names were standardized, and only relevant fields were retained. The national aggregate row was removed so that the dataset reflected state-level observations only.

After cleaning, the two datasets were merged on state code and year. Electricity sales values were converted from megawatt-hours (MWh) to kilowatt-hours (kWh), and per-capita residential electricity consumption was calculated by dividing total electricity consumption by state population for each year. 

import pandas as pd

# --- Load and clean population data ---
pop = pd.read_excel("USCensus_PopbyState.xlsx", header=1)
pop = pop.rename(columns={pop.columns[0]: "state"})
pop["state"] = pop["state"].str.lstrip(".")

pop_long = pop.melt(
    id_vars="state",
    var_name="year",
    value_name="population"
)
pop_long["year"] = pop_long["year"].astype(int)

state_to_code = {
    "Alabama":"AL","Alaska":"AK","Arizona":"AZ","Arkansas":"AR","California":"CA",
    "Colorado":"CO","Connecticut":"CT","Delaware":"DE","District of Columbia":"DC",
    "Florida":"FL","Georgia":"GA","Hawaii":"HI","Idaho":"ID","Illinois":"IL",
    "Indiana":"IN","Iowa":"IA","Kansas":"KS","Kentucky":"KY","Louisiana":"LA",
    "Maine":"ME","Maryland":"MD","Massachusetts":"MA","Michigan":"MI",
    "Minnesota":"MN","Mississippi":"MS","Missouri":"MO","Montana":"MT",
    "Nebraska":"NE","Nevada":"NV","New Hampshire":"NH","New Jersey":"NJ",
    "New Mexico":"NM","New York":"NY","North Carolina":"NC","North Dakota":"ND",
    "Ohio":"OH","Oklahoma":"OK","Oregon":"OR","Pennsylvania":"PA","Rhode Island":"RI",
    "South Carolina":"SC","South Dakota":"SD","Tennessee":"TN","Texas":"TX",
    "Utah":"UT","Vermont":"VT","Virginia":"VA","Washington":"WA",
    "West Virginia":"WV","Wisconsin":"WI","Wyoming":"WY"
}
pop_long["state_code"] = pop_long["state"].map(state_to_code)

# --- Load and clean electricity sales data ---
eia = pd.read_excel("EIA_sales_data.xlsx", header=1)
eia.columns = eia.columns.str.lower()
eia = eia[["year", "state", "sales_megawatthours"]]
eia = eia.rename(columns={"sales_megawatthours": "sales_mwh"})

# --- Merge and compute per-capita consumption ---
df = pop_long.merge(eia, on=["state", "year"], how="inner")

df["sales_kwh"] = df["sales_mwh"] * 1000
df["kwh_per_capita"] = df["sales_kwh"] / df["population"]

# --- Export final dataset ---
df_final = df[["state", "year", "population", "sales_mwh", "kwh_per_capita"]]
df_final.to_csv(
    "residential_kwh_per_capita_by_state_2010_2024.csv",
    index=False
)

The resulting normalized dataset was exported as a CSV file and used as the input for all subsequent visualizations in Tableau.

5.0 Findings and Limitations

Tableau sheets are used to highlight cross-state comparisons, long-term trends, and differences in the timing and magnitude of changes in residential electricity consumption per capita. These visuals and descriptions are designed to support discussion on electricity usage.

This analysis shows per-capita residential electricity consumption trends vary substantially across U.S. states, both in direction and timing. More specifically, much of the Southeast shows sustained declines in annual consumption per capita, while several states in the Intermountain West show increases over the same period.

While the results provide descriptive insight into how residential electricity consumption has evolved over time, this analysis does not attribute observed changes to specific causes. Factors such as weather variation, energy prices, housing characteristics, efficiency standards, and behavioral changes are outside the scope of the data used here. As a result, the findings should be interpreted as structural patterns rather than evidence of underlying drivers.

Future analyses could extend this work by incorporating weather-adjusted demand measures, energy price data, or housing stock characteristics to better contextualize state-level trends. Nonetheless, the current analysis demonstrates how careful normalization, comparative framing, and restrained interpretation can yield meaningful insights while maintaining analytical rigor and transparency.
